# self-improve

セッションをまたいだ**自己学習フィードバックループ**を作るプラグイン。
「記録 → 学習 → 次回適用」の3点を、トークンを使わないフック＋必要時のスキルで回します。

## 構成

| 要素 | イベント | 役割 | トークンコスト |
|---|---|---|---|
| `hooks/journal.sh` | Stop | セッションの git スナップショットを `~/.claude/session-journal/` に自動記録 | ゼロ (LLM 呼び出しなし) |
| `hooks/session-context.sh` | SessionStart (`startup`/`resume`) | 直近セッションの記録を冒頭の文脈に自動注入 | ゼロ |
| `skills/retro` | `/retro` | 振り返りで得た教訓を永続メモリ / CLAUDE.md へ書き戻す | 実行時のみ |

## 仕組み

```
            ┌─────────────────────────────────────────────┐
            │  セッション中の作業                          │
            └─────────────────────────────────────────────┘
                       │                         │
              Stop フック                  /retro (任意)
                       ▼                         ▼
   ~/.claude/session-journal/        ネイティブメモリ / CLAUDE.md
     (何をしたか・自動)                  (何を学んだか・厳選)
                       │                         │
                       └───────────┬─────────────┘
                                   ▼
                   次回 SessionStart で自動注入 / 想起
                   → AI が前回の続きから自走できる
```

- **journal.sh**: 応答が終わるたびに、作業中リポジトリのブランチ・直近コミット・未コミット変更を
  1セッション=1ファイルで上書き保存します。非 git ディレクトリでは何もしません。14日より古い記録は自動削除されます。
- **session-context.sh**: 新規/再開セッションの冒頭で、同じリポジトリ(無ければ全体)の直近3ファイルを
  読み込み、stdout に出力して文脈へ注入します。前回の続きを AI が把握した状態で始められます。
- **/retro**: ユーザーの訂正・好み、プロジェクト固有の落とし穴、再発した問題などの
  **再利用できる教訓**だけを抽出し、永続メモリに書き戻します。ネイティブメモリが無ければ `CLAUDE.md` にフォールバックします。

## 設計上の注意

- フックは停止を一切ブロックしません (常に `exit 0`)。`stop_hook_active` 中は二重記録を避けます。
- 記録先 `~/.claude/session-journal/` はリポジトリ外のユーザーホーム配下なので、コミット対象にはなりません。
- ネイティブの Claude Code メモリ機能とは**補完関係**です。フックが「自動・網羅・ゼロトークン」の作業記録を、
  `/retro` が「厳選した教訓」を担い、メモリ機能の想起と組み合わせてループを閉じます。

## 関連

- 同じ Stop イベントを使う他のフック(通知系など)とは独立して動作し、干渉しません。
- effort/モデルをタスク規模で調整したいときは `token-ops` の `effort-router` スキルを参照。
