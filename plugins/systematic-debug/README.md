# systematic-debug

**根本原因にたどり着くための体系的デバッグ**をガイドするプラグイン。
症状を場当たり的にパッチするのではなく、「理解 → 仮説 → 検証 → 修正」の4フェーズで原因を特定します。

## 構成

| スキル | 役割 |
|---|---|
| `/systematic-debug` | バグ・テスト失敗・CI 失敗の根本原因を、4フェーズの手順で追跡する |

## 使い方

```bash
# 問題を渡して起動
/systematic-debug CI の lint ジョブだけが落ちる

# 直近の文脈から推定
/systematic-debug
```

## 鉄則(Iron Law)

> **理解していない問題を修正してはならない。**

原因を説明できないまま当て推量で修正を重ねると、症状が移動するだけで再発します。本スキルは「なぜそうなるか」を言語化してから直すことを強制します。

## 関連

- タスク規模に応じた effort/モデルの選定は `effort-router` を参照(不確実性が高い問題は xhigh 推奨 → 本スキルで原因追跡)。
- 原因特定後の修正をテスト先行で進めたい場合は `tdd-cycle` プラグインを参照。
