# tdd-cycle

**テスト駆動開発(TDD)のサイクルを強制**するプラグイン。
RED(失敗するテストを先に書く)→ GREEN(最小実装で通す)→ REFACTOR(整える)を、順序を崩さずに回します。

## 構成

| スキル | 役割 |
|---|---|
| `/tdd-cycle` | 実装したい機能を渡すと、RED → GREEN → REFACTOR の3フェーズで進める |

## 使い方

```bash
/tdd-cycle ユーザー名のバリデーション(空文字を弾く)を追加
```

## 鉄則(Iron Law)

> **テストを書く前に実装コードを書いてはならない。**

まず「期待する振る舞い」を失敗するテストとして表現し、それを通す最小実装を書く。テストは仕様であり、後付けの飾りではありません。
